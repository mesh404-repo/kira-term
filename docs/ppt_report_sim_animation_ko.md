# TBS 공정 시뮬레이션/애니메이션 연계 보고서 (PPT 초안)

## 1장. 시뮬레이션 구성 및 운영 기준

### 시뮬레이션 구성
- 기반 엔진: `simpy` 시간 이벤트 기반 공정 시뮬레이션
- 대상 자원: OHT, BP1~BP4, EP1~EP3
- 공정 단계: 투입 -> 이송 -> 공정 -> 반출
- 이벤트 표준: `EAPEIS_PORT_*` XML 생성/역파싱 기준

### simpy 기반 시뮬레이션 운영 개념
- 공장이 실제 시간을 사용해 동작하는 방식을 가상 환경에서 재현하는 구조.
- LOT 이송에 8초가 필요하면 화면과 로그도 8초 경과 후 다음 단계로 전환.
- 공정 시간 6초, 애니메이션 10초인 경우 10초 종료 시점까지 대기 후 다음 단계로 전환.
- 이벤트 즉시 완료 방식이 아니라 순서/대기/전환이 포함된 시간 기반 진행 유지.
- 시간 로그, 단계 전환, 애니메이션 실행 순서를 동일한 시나리오 기준으로 정합성 있게 운영.

### 운영 규칙(현재 반영)
- EP에 빈 포트가 있으면 OHT가 EP로 직접 투입
- EP가 모두 찬 경우 BP 경유(BP1 -> BP2/3/4) 경로 사용
- `OHT -> BP1` 구간은 `ARRIVED` 이벤트 기준으로 처리
- 애니메이션 시간이 공정보다 길면 애니메이션 종료까지 다음 공정 대기

### 운영 규칙 반영 방식(코드 흐름 요약)
- 시뮬레이션 시작 시 LOT 큐 준비 후 직렬 공정 루프 진입.
- 루프 내 우선순위: EP 빈 포트 확인 -> OHT->EP 직접 투입 수행.
- EP가 모두 점유 상태면 BP1 경유 투입 후 BP2/3/4 버퍼 이송 수행.
- 버퍼 LOT 존재 + EP 빈 포트 조건 만족 시 BP->EP 이송 후 즉시 공정 단계 진입.
- 공정 완료 후 EP->OHT 반출, 포트 상태 갱신, 완료 LOT 누적.
- 각 단계에서 이벤트(XML 기준)와 애니메이션 실행을 연계하고, 애니메이션 종료 시점 기준으로 다음 단계 진행.

### 확인 가능한 운영 정보
- 진행현황: 현재 단계, 예상 시간, 진행률
- 이력로그: 공정 이벤트/스토리 로그
- 애니메이션 실행이력: 이벤트, 매핑 파일명, 실행 상태(START/DONE)

### 권장 캡처
- [캡처-1] 시뮬레이션 제어창(진행현황/이력로그/애니메이션 실행이력 동시 표시)
- [캡처-2] 공정확인 Alert(XML + 매핑 JSON 파일명 + 파일 존재 여부)

## 2장. 공정-애니메이션 연계 흐름

### 동작 흐름
1. 공정 이벤트 발생(`simpy` 시간 흐름 기반)
2. XML 생성(`sequence_name`, `port_id`, `from/to_port_id`)
3. XML 역파싱 후 표준 payload 구성
4. 현재 공정 상황에 맞는 애니메이션 JSON 선택
5. 애니메이션 실행(큐 직렬 처리로 끊김/중복 최소화)
6. 애니메이션 종료 후 다음 공정으로 자연스럽게 전환

### 동작 흐름 설계시 중요사항
- "무엇을 할지 결정" -> "신호(XML)로 표현" -> "해당 장면(JSON) 실행"의 반복 구조.
- 시뮬레이션은 공정 판단 담당, 애니메이션은 판단 결과를 시각 장면으로 표현.
- 동일 공정이라도 포트 상태가 다르면 서로 다른 장면으로 분기 가능.

### 연계 실행 핵심 포인트
- 공정 시간과 애니메이션 시간을 비교해 긴 쪽 기준으로 다음 단계 진행
- 포트 상태(비어 있음/점유)에 따라 이송 경로가 달라지고, 해당 경로에 맞는 애니메이션 실행
- 동일 공정 내에서도 이벤트 유형(도착/이송/반출)에 맞춰 동작이 연속적으로 연결

### 권장 캡처
- [캡처-3] 이벤트 로그 일부(XML 생성/역파싱/매핑 경로)
- [캡처-4] 애니메이션 실행이력(이벤트별 파일명, 실행 상태)
- [캡처-5] 공정 진행 중 장면 연속 캡처(이송 -> 도착 -> 공정)

## 3장. 운영 결과 확인 및 보고 산출물

### 결과 확인 방식
- 실시간 UI 로그 3종으로 단계별 추적 가능
- 공정확인 창에서 XML/실행 애니메이션 파일을 함께 확인 가능
- 애니메이션 실행이력으로 이벤트별 실행 상태(START/DONE) 추적 가능

### 산출물
- 시뮬레이션 종료 시 Excel 자동 생성(`sim_logs_YYYYmmdd_HHMMSS.xlsx`)
- 시트 구성: 진행현황 / 이력로그 / 애니메이션실행이력
- 저장 정렬: 3개 시트 모두 시간 순(먼저 실행된 항목이 상단)

### 권장 캡처

- [캡처-7] 애니메이션실행이력 시트(이벤트-파일명-상태 컬럼)
