## 슬라이드 1) 시스템 개요 및 제공 가치 (간부 보고용 요약)

### 목적(한 문장)
- **설비 신호 기반 공정 흐름을 “시간 축으로 재현”하고, 동일 이벤트를 “로그 + 3D 동작”으로 일관되게 검증/시연하는 시뮬레이션 플랫폼**

### 핵심 제공 기능(요약)
- **시뮬레이션 엔진(simpy)**: LOT 이동/공정/회수 등 이벤트를 설정 가능한 시간 모델로 재생
- **가시화(운영 관점)**: 진행현황(실시간), 이력로그(스토리), 애니메이션 실행이력(증적) 제공
- **검증 제어**: “각 공정 확인” 옵션으로 단계 시작 전 XML/예상 시간 확인 후 진행(감사/검증 용이)
- **산출물 자동화**: 종료 시 로그 3종을 **엑셀(xlsx) 3개 시트**로 자동 저장(보고/공유 즉시 가능)

### 기대 효과(간부 관점)
- **현장 커뮤니케이션 비용 감소**: 텍스트 로그 + 3D 동작으로 의사소통 정합성 향상
- **변경 영향 최소화**: 이벤트→XML→룰 기반 매핑으로, 공정/동작 변경을 “설정 파일 + JSON” 중심으로 흡수
- **확장성**: 이벤트 종류/설비/포트 수 변화에 대해 모듈 단위로 확장 가능

## 슬라이드 2) 운영 시나리오(순서)와 애니메이션 연결 방식 (간부 보고용 요약)

### 공정 시나리오(현재 구현: 직렬 1 LOT 기준)
1) **OHT → BP1** (LOT 투입)
2) **BP1 → BP2/3/4** (버퍼로 이동)
3) **BP(버퍼) → EP(공정 포트)** (공정 투입)
4) **EP 공정 수행** (metro_time)
5) **EP → OHT 회수** (완료)

### 애니메이션 연결(표준화된 파이프라인)
- **이벤트 발생** → **XML 생성** → **XML 역파싱(정식 이벤트/포트 정보 표준화)** → **룰 기반 매핑** → **JSON 시퀀스 실행**
- 매핑 우선순위:
  - `config/event_animation_rules.json` (상태 기반, 우선 적용)
  - `config/event_animation_map.json` (단순 매핑, fallback)

### 운영/보고 관점의 “증적(Traceability)”
- 애니메이션 실행이력에는 **실행 시각(시뮬레이션 시간)**, **예상 소요시간**, **실제 소요시간**, **실행 파일/동작 설명**이 남습니다.
- 종료 시 엑셀 시트(애니메이션실행이력)는 **열(column) 분리**로 저장되어, 사후 분석/보고서 편집이 쉽습니다.
